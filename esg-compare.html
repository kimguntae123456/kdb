두 가지 접근, 한 번에 비교하기

기존 페이지의 내용은 그대로 두되, 스타일만 바꾼 A방식과 HTML 구조까지 정돈한 C방식으로 나누어 작업했습니다. 두 페이지를 직접 열어보고 가독성·구조감을 비교해보세요.

방식 A

스타일만 교체

원본 HTML은 그대로 두고 새 CSS 한 장(styles-v2.css)으로 시각적 위계와 본문 가독성만 끌어올렸습니다. 가장 빠르고 안전한 방법이지만, 시맨틱이 약한 마크업의 한계는 그대로 남습니다.

  • 본문 글꼴을 Noto Serif KR 기반으로 통일, 행간 1.9
  • 소제목 4단계(section-h · para-title · sub-title · tag-title) 위계 명확화
  • 강조색 1개로 정돈, 카테고리 색을 모두 동일 톤으로 통일
  • HTML은 단 한 줄도 수정하지 않음 — 기존 코드와 100% 호환

방식 C

HTML 구조까지 정돈

A의 시각 개선에 더해, 마크업 자체를 의미 있는 시맨틱 태그로 바꿨습니다. CSS는 클래스가 아니라 태그를 직접 가리키므로 단순해지고, 스크린리더·검색엔진에도 더 잘 읽힙니다.

  • 용어 박스를 <dl><dt><dd> 정의 목록으로 — 묶음으로 시각적 일체감
  • 예상 꼬리질문을 <details><summary> — 클릭해서 펼쳐보는 인터랙션
  • 헤딩 위계 정상화: h2 > h3 > h4 > h5 (이전엔 h4가 글 제목)
  • 빈 헤딩·중복 제목 일괄 제거, 글 헤더는 <header> 시맨틱으로
C방식 페이지 열기

주요 변환 매핑 (A → C)

요소 A방식 (기존 마크업 유지) C방식 (시맨틱 변환)
글 헤더 <h4 class="ia-tag-title"><span class="tag-label">…</span>…</h4> <header class="post-head"><span class="post-tag">…</span><h2>…</h2></header>
대단원 소제목 <div class="ia-section-h">…</div> <h3 class="post-section">…</h3>
단락 소제목 <h3 class="ia-para-title">…</h3> <h4 class="post-h4">…</h4>
① ② 항목 <h4 class="ia-sub-title">…</h4> <h5 class="post-h5">…</h5>
용어 박스 <div class="term-box"><span class="term-name">…<span class="term-def">…</div> <dl class="terms"><dt>…</dt><dd>…</dd></dl>
예상 꼬리질문 <div class="qa-q">Q</div><div class="qa-a">A</div> <details class="qa"><summary>Q</summary><div>A</div></details>
빈 헤딩 그대로 (<h3></h3> 다수 잔존) 일괄 제거
